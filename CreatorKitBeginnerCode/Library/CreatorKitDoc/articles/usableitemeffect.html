    Writing new Usage Effect

    A Usage Effect is a script that can be added to a Usable Item and allow to specify what happen when the item is double clicked in the inventory.

    We provide a shortcut to create a pre-written script that implement a Usage Effect. Use the menu entry Beginner Code > Create Item Effect.

    In the popup that open, choose a name (no space, no special character, the popup will stop you from validating a non-valid name) and press Ok.

    This will create the new script in the Assets/Scripts/ItemEffect folder, and should select it. You can double click to open it, and it will look like this :

    using System.Collections;
using System.Collections.Generic;
using UnityEngine;
using SceneKitCode;

public class EFFECTNAME : UsableItem.UsageEffect
{
    public override bool Use(CharacterData user)
    {
        return false;
    }
}

    With EFFECTNAME being the name you entered in the popup earlier.

    Now to write a new effect, you just need to write it in the Use function. The user parameter will have the CharacterData of the character that used that Item (in the case of our game, the player as it is the only character that can use object in current implementation). Through that you can access stats, inventory & equipmentite to modify them (see the Api Documentation on that website for more info on the CharacterData class)

    This function return a boolean, you should return :

    • true if the object was indeed used, it will be "consumed" and removed from the ivnentory (if it's a stack of objects, the count is decremented by 1, if it's a single object, it is removed)
    • false if the object couldn't be used (e.g. you can test if health is already at maximum before adding health, and return false if it is so the object can't be used when health is already at max)
