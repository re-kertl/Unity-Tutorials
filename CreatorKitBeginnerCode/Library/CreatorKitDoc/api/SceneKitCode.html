    Namespace SceneKitCode

    Classes

    BaseElementalEffect

    Base class to derive to write you own custom Elemental effect that can be added to a StatsSystem. There is a default implementation called ElementalEffect that can be used to make Physical/Fire/Electrical/Cold damage across time

    Derived class must implement the Equals function so we can check if 2 effects are the same (e.g. the default implementation ElementalEffect will consider 2 effect equal if they do the same DamageType)

    CharacterData

    This define a character in the game. Character is use in a loose sense, as it just mean something that can be attack and have some stats including health. It can also be an inanimate object like a breakable box.

    Container

    Special InteractableObject that will trigger the LootSpawner on it when interacted with, and delete itself (a container can only be looted once)

    ElementalEffect

    Default implementation of the BaseElementalEffect. The constructor allow to specify what type of damage is done, how much is done and the speed (time) between each damage (by default 1, every second).

    EquipmentSystem

    Handle the equipment stored inside a CharacterData. Will take care of unequiping the previous item when equiping a new one in the same slot.

    Helpers

    Helper class containing diverse functions that avoid redoing common things

    HighlightableObject

    All object than can be highlighted (enemies, interactable object etc.) derive from that class, which take care of Setting the material parameters for it when it get highlighted. (if the object use another material, it will just ignore all the changes)

    InteractableObject

    InventorySystem

    This handle the inventory of our character. The inventory is composed of 32 slot maximum, each slot can hold one TYPE of object, but those can be stacked without limit (e.g. 1 slot used by health potion, but own 20 health potions)

    InventorySystem.InventoryEntry

    Loot

    Describe an InteractableObject that can be picked up and grant a specific item when interacted with.

    It will also play a small animation (object going in an arc from spawn point to a random point around) when the object is actually "spawned", and the object become interactable only when that anim is finished.

    Finally it will notify the LootUI that a new loot is available in the world so the UI display the name.

    SpawnPoint

    When the player walk through the trigger on the same GameObject as that script, the GameObject position become the new place where the player will respawn when their health reach 0.

    StatSystem

    This handle the stats of a CharacterData. It store the health and strength/agility/defense stats. This class contains various functions to interact with stats, by adding stat modification, elemental effect or damages.

    StatSystem.StatModifier

    Can be added to a stack of modifier on the StatSystem to modify the value of the base stats e.g. a weapon adding +2 strength will push a modifier on the top of the stack

    They have 2 mode : absolute, where value are added as is, and Percentage, where value are converted to percentage (e.g. a value of 50 in strength in a Percentage modifier will increase the strength by 50%)

    StatSystem.Stats

    StatSystem.TimedStatModifier

    This is a special StatModifier, that get added to the TimedStatModifier stack, that will be automatically removed when its duration reach 0. Contains a classic StatModifier that is actual modification.

    VFXDatabase

    This allow to define a list of VFX prefab with a name. An editor script take care of generating a C# file containing an enum with the given name to index the Entries array so you can do in code something like VFXManager.SpawnFX(MyEffectName)

    VFXDatabase.VFXDBEntry

    VFXManager

    This script handle displaying VFX in the game. One instance should be in the scene, and is part of the Manager prefab It will create pools of the VFX prefabs defined in the given Database, and through the GetVFX/PlayVFX allow to get an instance at a given position. The pool are rotating queue, so when you get a new instance, that instance will be move the back of the available instance queue

    VFXManager.VFXInstance

    On instance of VFX. Effect is the actual GameObject of the effect you can disable once finish. Index is used by the manager to know which effect this instance is part of.
